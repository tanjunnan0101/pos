# Add centiliter (cl) volume unit

## GitHub Issues
- **Issue:** https://github.com/tanjunnan0101/pos/issues/214
- **214**

## Problem / goal

Support **centiliter** (`centiliter` / **cl**) as a volume unit across backend and frontend so staff can enter and convert beverage quantities in cl (common for wine/spirits). Conversion: **1 cl = 10 ml** (base volume unit remains milliliter).

## High-level instructions for coder

- **Backend enum and conversions:** Add `centiliter` to `UnitOfMeasure` in `back/app/inventory_models.py` and `UNIT_CONVERSIONS` with factor **10** ml per cl (same volume base as `milliliter`). Ensure `convert_units` and any pricing/inventory paths that enumerate units accept the new value.
- **Frontend type and lists:** Extend `UnitOfMeasure` in `front/src/app/inventory/inventory.types.ts`. Include `centiliter` in volume unit arrays used by inventory items, purchase orders, and **`pricing-helper.component.ts`** unit dropdowns (container/serving).
- **i18n:** Add `INVENTORY.UNITS.CENTILITER` (and equivalent key style) to **all** `front/public/i18n/*.json` locale files; keep keys in sync across languages.
- **API / OpenAPI:** If unit lists are exposed from `inventory_routes` or similar, confirm the new enum appears without breaking existing clients.
- **Tests:** Add or extend backend tests for `convert_units` involving cl ↔ ml (and cl ↔ liter if useful). Smoke: open pricing helper or inventory unit picker and confirm **cl** appears with correct label.
- **Out of scope unless required:** DB migration only if persisted enums/constraints exist beyond string columns; follow existing patterns for other volume units.

## Implementation notes

- Backend: `UnitOfMeasure.centiliter` with `UNIT_CONVERSIONS` factor **10** (ml base).
- Frontend: type + unit pickers in inventory items, purchase orders, pricing helper.
- i18n: `INVENTORY.UNITS.CENTILITER` in all nine locale files.
- Tests: `back/tests/test_inventory_unit_conversion.py` (cl ↔ ml, cl ↔ liter, volume type).
- Inventory API unit list iterates `UnitOfMeasure` enum — **centiliter** included automatically.
- **DB:** `back/migrations/20260528140000_add_centiliter_unitofmeasure.sql` — `ALTER TYPE unitofmeasure ADD VALUE IF NOT EXISTS 'centiliter'`.
